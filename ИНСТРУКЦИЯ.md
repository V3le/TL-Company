# Инструкция по запуску сервера через Node.js (вручную)

## Быстрый запуск

### В терминале (CMD / PowerShell / VS Code):
```bash
cd C:\Users\v3lee\Desktop\tl-web\public
npx --yes live-server --port=8000 --open=/
```

Или одной командой:
```bash
cd C:\Users\v3lee\Desktop\tl-web\public && npx --yes live-server --port=8000 --open=/
```

---

## Подробная инструкция

### Шаг 1: Открыть терминал

**Windows CMD:**
- Нажмите `Win + R`
- Введите `cmd` и нажмите Enter

**PowerShell:**
- Нажмите `Win + X`
- Выберите "Windows PowerShell" или "Терминал"

**VS Code / Cursor:**
- Нажмите `Ctrl + ~` (тильда)
- ИЛИ меню: Terminal → New Terminal

---

### Шаг 2: Перейти в папку public

```bash
cd C:\Users\v3lee\Desktop\tl-web\public
```

**Проверка:**
Убедитесь, что вы в правильной папке:
```bash
dir
```
Должны увидеть файлы: `index.html`, `css/`, `js/`, `images/`

---

### Шаг 3: Запустить сервер

```bash
npx --yes live-server --port=8000 --open=/
```

**Что происходит:**Ё
- `npx` - запускает пакет Node.js
- `--yes` - автоматически устанавливает live-server если нужно
- `live-server` - название пакета
- `--port=8000` - порт сервера
- `--open=/` - автоматически откроет браузер

---

## Альтернативные команды

### Если npx не работает, используйте полный путь:
```bash
"C:\Program Files\nodejs\npx.cmd" --yes live-server --port=8000 --open=/
```

### Запуск без автоматического открытия браузера:
```bash
npx --yes live-server --port=8000
```
Затем откройте вручную: http://localhost:8000

### Запуск на другом порту:
```bash
npx --yes live-server --port=3000 --open=/
```
Сервер будет на: http://localhost:3000

### Запуск с задержкой обновления:
```bash
npx --yes live-server --port=8000 --open=/ --wait=500
```
`--wait=500` - задержка 500мс перед обновлением страницы

---

## Остановка сервера

Нажмите `Ctrl + C` в терминале, где запущен сервер.

---

## Проверка Node.js

Если команды не работают, проверьте установку Node.js:

```bash
node --version
```

Должно показать версию (например: v25.0.0)

Если не работает, Node.js не установлен или не в PATH.

---

## Решение проблем

### Ошибка: "npx не является внутренней или внешней командой"

**Решение 1:** Используйте полный путь:
```bash
"C:\Program Files\nodejs\npx.cmd" --yes live-server --port=8000 --open=/
```

**Решение 2:** Добавьте Node.js в PATH:
1. Откройте "Переменные среды"
2. Добавьте `C:\Program Files\nodejs\` в системную переменную PATH
3. Перезапустите терминал

### Ошибка: "Порт 8000 уже занят"

Измените порт:
```bash
npx --yes live-server --port=8001 --open=/
```

### Ошибка: "Папка public не найдена"

Убедитесь, что вы в правильной директории:
```bash
# Посмотреть текущую папку
cd

# Перейти в папку проекта
cd C:\Users\v3lee\Desktop\tl-web\public

# Проверить содержимое
dir
```

## Полезные команды

### Установить live-server глобально (опционально):
```bash
npm install -g live-server
```
После этого можно запускать просто:
```bash
live-server --port=8000 --open=/
```

### Проверить установленные глобальные пакеты:
```bash
npm list -g --depth=0
```

### Посмотреть все параметры live-server:
```bash
npx --yes live-server --help
```

## Автообновление страницы

При использовании `live-server` страница **автоматически обновляется** при изменении файлов.

**Как это работает:**
1. Вы редактируете файл (например, `index.html`)
2. Сохраняете файл (`Ctrl + S`)
3. Страница в браузере обновляется автоматически
4. Не нужно нажимать F5!
